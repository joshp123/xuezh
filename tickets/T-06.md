    # T-06: SRS primitives v1: review start/grade + due/preview

    ## Goal
    Implement mechanical review flow with stored outcomes and due dates, without pedagogical recommendations.

    ## Deliverables
    - `chlearn review start --limit N` returns due items
- `chlearn review grade --item ITEM --grade 0..5 [--next-due ISO | --rule sm2|leitner]`
- `chlearn report due --limit N`
- `chlearn srs preview --days D` (counts by day)


    ## Acceptance criteria
    - Due items ordered deterministically: due_at ASC, item_id ASC
- Grade records append-only review_event
- Scheduling is mechanical:
  - either engine applies rule chosen by model
  - or model supplies exact next_due override
- No “what next” output fields


    ## Test plan (RGR)
    - Unit: schedule transform functions
- E2E: create due item -> start -> grade -> confirm due_at updated
- Integration: CLI JSON output validates schema
